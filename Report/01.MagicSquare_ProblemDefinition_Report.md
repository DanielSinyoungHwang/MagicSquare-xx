## 1) 주제 한 문장

> **4×4 부분 마방진(빈칸 2, 합 34)을 풀었을 때, 행·열은 맞다고 믿지만 대각선 조건을 늦게 확인해 틀린 판정을 늦게 고치는 비용을 줄인다.**

- 솔루션 최소화: “앱”, “자동 풀이”, “Solver”, “힌트” 언급 없음
- Mom Test 초점: **검증 누락(주대각선)** + **늦은 재판정(~20분)** — “못 풀어서”가 아님

## 2) R-G-I-O

| | 내용 |
|---|------|
| **Role** | 4×4 부분 마방진을 손·코드·ECB로 다루는 **학습자** — 채운 격자가 맞는지 스스로 확인하는 사람 |
| **Goal** | 채운(또는 거의 채운) 4×4 격자에 대해 **행 4 + 열 4 + 대각선 2** 조건을 빠짐없이 점검하고, **어느 조건이 깨졌는지** 즉시 알 수 있다 |
| **Input** | 4×4 정수 격자. 값域: `0`(빈칸, 최대 2) 또는 `1~16`(각 1회). ECB 실습·손풀이 결과판 |
| **Output** | 조건별 **통과/실패**. 실패 시 **행·열·대각선 중 어디**가 합 34가 아닌지 (실제합·기대합) |

---

## 3) 성공 기준 3개 (Mom Test 증거 연결)

| # | 성공 기준 | 연결된 증거 |
|---|-----------|-------------|
| **S1** | 4×4 격자 1판에 대해 **행·열·대각선 10개 조건**을 **한 번에** 점검할 수 있다 | ① ECB 작업 **어려움** — 확인·검증 단계에 시간·노력 과다 |
| **S2** | 실패 시 **“main_diagonal sum=32 expected=34”**처럼 **어느 조건이 틀렸는지** 명시적 출력 | ② **20분 뒤 마지막에** 알았던 대각선 누락 → 조건별 즉시 식별 |
| **S3** | 행·열만 맞고 **주대각선(또는 부대각선)이 틀린** 케이스에서 **즉시** NG를 잡는다 (20분 후가 아님) | ③ 행·열 OK 착각 → **검증 중** 대각선 NG **즉시** 재현·차단 |

**세션 3 Done:** S1·S2·S3을 Test Loop로 자동 검증 가능할 때.

## 4) 표면 문제 — 이번 프로젝트에서 하지 않을 것

| 표면 문제 (솔루션 섞임) | 하지 않는 이유 |
|-------------------------|----------------|
| 4×4 마방진 **자동 풀이·Solver·힌트** | 진짜 문제는 풀이 실패가 아니라 **검증 누락·늦은 재판정** |
| **계산기·34 맞추기 도우미** | 20분 중 긴 구간(빈칸·손계산)은 범위 밖; **검증**만 다룸 |
| **검증 자동화 CLI**를 제품 슬로건으로 | CLI는 구현 수단일 뿐; 주제는 **조건 전수 점검** |
| **튜토리얼·설명 UI·강의 앱** | Mom Test 표면(설명 어려움) — v0.1 Out of Scope |
| **3×3, 5×5, n×n** 일반화 | 페르소나는 **4×4, 빈칸 2, 합 34**만 |
| **문제 생성·게임화·공유·SNS** | 검증과 무관 |
| **예쁜 GUI·모바일 앱** | 세션 3는 Rule·Command·Test Loop 우선 |
